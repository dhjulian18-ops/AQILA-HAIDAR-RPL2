Latar Belakang & Deskripsi Masalah

Mengapa aplikasi ini dibutuhkan?

🏢 Kondisi Saat Ini

Sebuah tempat olahraga memiliki beberapa admin lapangan. Setiap admin bertugas mencatat data booking yang masuk secara manual, sehingga rentan terjadi kesalahan dan tumpang tindih data.

🔒 Masalah Keamanan Data

Data booking milik satu admin tidak boleh terlihat, diedit, atau dihapus oleh admin lain. Diperlukan sistem isolasi data per pengguna.

📋 Solusi yang Dibutuhkan

Aplikasi web terpusat dengan autentikasi, CRUD data booking, dan filter data berdasarkan akun admin yang sedang login.

Tujuan Proyek

1

Pengelolaan Booking Digital

Mencatat data booking lapangan olahraga secara terstruktur dan aman.

2

Autentikasi Admin

Register dan login khusus untuk masing-masing admin lapangan.

3

Isolasi Data Per Admin

Admin hanya melihat dan mengelola data booking miliknya sendiri.

4

Laporan yang Dapat Dicetak

Admin bisa mencetak laporan booking untuk keperluan administrasi.

Teknologi & Tools yang Digunakan

Stack teknologi dalam pengembangan aplikasi

🐘

PHP Native

Backend tanpa framework — lebih mudah dipahami dan dikontrol penuh

🗄️

MySQL

Database relasional untuk menyimpan data user dan booking lapangan

🎨

Bootstrap 5

CSS framework untuk tampilan responsif, rapi, dan modern

🖥️

XAMPP

Web server lokal Apache + MySQL untuk menjalankan aplikasi di komputer

Keunggulan Teknologi Pilihan

  • PHP Native mudah dipelajari dan tidak perlu instalasi framework
  • MySQL gratis, stabil, dan terintegrasi dengan XAMPP
  • Bootstrap 5 membuat tampilan otomatis responsif di HP & laptop
  • Bootstrap Icons untuk ikon tanpa perlu gambar tambahan
  • MySQLi Prepared Statements untuk keamanan query database
  • Session PHP untuk manajemen autentikasi yang sederhana

Fitur Keamanan

  • password_hash() — password tidak disimpan polos
  • Prepared Statements — cegah SQL Injection
  • htmlspecialchars() — cegah serangan XSS
  • Session Check — proteksi semua halaman
  • user_id filter — isolasi data per admin
  • WHERE id AND user_id — amankan edit & hapus

Desain Database

Struktur tabel dan relasi antar data

📋 Tabel: users

FieldTipeKeterangan
idINT (PK)ID unik auto increment
nameVARCHAR(100)Nama lengkap admin
emailVARCHAR(100)Email unik untuk login
passwordVARCHAR(255)Password ter-hash (bcrypt)
created_atTIMESTAMPWaktu akun dibuat
updated_atTIMESTAMPWaktu akun diperbarui

📋 Tabel: booking

FieldTipeKeterangan
idINT (PK)ID unik booking
user_idINT (FK)🔗 Relasi ke tabel users
nama_pemesanVARCHAR(100)Nama pelanggan
jenis_lapanganVARCHAR(100)Badminton, Futsal, dll
tanggal_bookingDATETanggal booking
jam_mulaiTIMEJam mulai main
jam_selesaiTIMEJam selesai main
total_bayarINTJumlah pembayaran
status_pembayaranVARCHAR(50)Lunas / DP / Belum Lunas
🔗
Relasi Database

Tabel booking.user_id → FOREIGN KEY → users.id  |  Setiap data booking terhubung langsung dengan admin yang membuatnya. Saat admin login, sistem otomatis memfilter data hanya milik admin tersebut.

🔐 Implementasi Keamanan Data

Menampilkan Data (SELECT)
SELECT * FROM booking
WHERE user_id = $_SESSION['user_id']
ORDER BY id DESC;
Mengamankan Edit/Detail
SELECT * FROM booking
WHERE id = $id
AND user_id = $_SESSION['user_id'];
Mengamankan Hapus
DELETE FROM booking
WHERE id = $id
AND user_id = $_SESSION['user_id'];
